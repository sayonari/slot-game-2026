# KNOWLEDGE - ドメイン知識・調査結果

## 業務・ドメイン知識
- ビデオスロットの基本構成：5リール×3段、20ペイライン、左端から3個以上の連続一致で当選。WILDは代替、SCATTERは位置不問でボーナス契機
- パチスロ的な「煽り」文法：オーラ色（青<緑<赤<虹）で期待度を示し、たまにガセ（外れでも赤）を混ぜることで毎回ドキドキさせる
- 中毒性の設計：天井（連敗救済）、プログレッシブジャックポット、再突入で育つマルチプライヤー、コレクション要素の組み合わせが有効

## 調査・リサーチ結果
- 効果音・BGMはWeb Audio APIの合成で実用十分な品質が出せる（素材収集不要・著作権リスクゼロ）
- BGMはルックアヘッド型ステップシーケンサ（setInterval 90ms + 0.22s先行スケジュール）で安定再生できる

## 技術的な知見
- Three.jsのリールは「円筒」より「平面5枚のスクロール＋傾き・奥行きで疑似ドラム」が確実で見た目も良い（テクスチャUV回転の罠を回避）
- 回転ブラーは事前生成した縦スメア版CanvasTextureに差し替えるだけで高速かつ効果的
- リール窓のDOMフレームは three.js の Vector3.project() で画面座標に投影して位置合わせする
- ブラウザの自動再生制限：AudioContextは初回 pointerdown/keydown で ensure() + resume() する
- Vite + GitHub Pages は base: '/リポジトリ名/' の設定が必須
- プレイデータは cookie より localStorage が適切（容量・API）。エクスポート/インポートは btoa(encodeURIComponent(JSON))
- Playwright検証時、ESMスクリプトは node_modules のあるディレクトリ配下に置く必要がある（importはスクリプトの場所基準で解決）

## 決定事項と理由
- ハイブリッド型（ビデオスロット基盤＋パチスロ演出）：ユーザー選択
- 素材はコード生成中心：外部依存なし・著作権クリーン・静的サイト完結のため（ユーザー選択）
- リポジトリはパブリック化してGitHub Pages公開：ユーザー承認済み
- RTPは config.ts の paytable / weight で調整可能な設計。娯楽用（換金なし）のため救済を厚めに設定

## 第2サイクルの知見（2026-06-12）
- 当たり率の調整は「ハズレ時の引き直し回数（rerolls）」が最も安全で効果的（配当表・重みを大きく崩さない）。rerolls=1で当たり率 約47%→72%
- エンジンの統計検証は Playwright の page.evaluate から Vite の開発サーバー経由で `/src/core/engine.ts` を動的importすると、ビルド不要で2万回シミュレーションできる
- 演出フローの目視検証は BONUS_CHANCE を一時的に0.95へ書き換え→HMR反映→Playwrightでセレクタ待ち（#bonus:not(.hidden) 等）が手早い。検証後必ず戻す
- ボーナス期待値はEV=Σ(weight×mult)/突入分母で見積もる。本作はEV≒1.7×ベット/スピンの大盤振る舞い設計（娯楽優先・換金なしのため意図的）
